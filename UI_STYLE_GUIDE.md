# UI_STYLE_GUIDE.md

## Design Goal
Build a modern, clean, professional support dashboard that feels fast, polished, and easy to use.

## Visual Style
- Clean layouts
- Rounded corners
- Soft shadows
- Subtle borders
- Balanced spacing
- Modern neutral palette with accent colors
- Clear contrast for readability

## Interaction Style
- Use hover effects on cards, buttons, table rows, and menu items.
- Use pointer cursor for clickable elements.
- Use smooth transitions for hover, focus, and state changes.
- Add gentle motion for panels, cards, and page entry where appropriate.
- Use loading states and skeletons for async content.

## 3D / Depth Feel
Use subtle depth effects instead of heavy decoration:
- Lift cards slightly on hover
- Add layered shadows
- Use gradients sparingly
- Keep the interface elegant, not flashy

## Responsive Behavior
- Desktop: multi-column layout
- Tablet: simplified panels
- Mobile: single-column layout with compact spacing
- Navigation should remain easy on smaller screens

## Typography
- Use strong hierarchy
- Headings should be clear and bold
- Body text should be readable and not too small
- Labels should be concise

## Dashboard Cards
- Use summary cards for tickets, complaints, SLA, and active users
- Add icons where useful
- Keep numbers easy to scan

## Tables
- Table rows should highlight on hover
- Status values should use badges
- Actions should be grouped cleanly

## Forms
- Use clear labels
- Show validation messages immediately
- Keep form spacing comfortable
- Make buttons obvious and easy to click

## Ticket Experience
- Ticket list must be easy to filter and scan
- Ticket detail should show conversation, metadata, and status clearly
- Status changes should feel immediate and visible

## Color Guidance
Use a modern palette such as:
- Base: white, slate, gray
- Accent: blue, indigo, cyan, or purple
- Success: green
- Warning: amber
- Error: red

## Accessibility
- Buttons should have visible focus states
- Text should remain readable on all backgrounds
- Avoid using color alone to convey meaning

## UI Principle
The interface must feel like a premium support CRM: clear, calm, fast, and trustworthy.
